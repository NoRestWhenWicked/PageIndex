# Skills

A **skill** is a repeatable, parameterized task you've taught the agent to do
well — a named recipe with inputs, steps, and an output shape. Skills are how the
second brain compounds: build one a week and the system gets more capable over time.

## Anatomy of a skill
- **Trigger:** what I say to invoke it (e.g. "summarize this meeting").
- **Inputs:** what it needs from me or the vault.
- **Steps:** the procedure the agent follows.
- **Output:** where the result goes and in what template.

## Building a new skill
1. Notice a task you repeat or want consistent.
2. Write it up using the structure above (copy an existing file).
3. Reference the right template from `Templates/`.
4. Use it, then refine the steps based on what went wrong.
