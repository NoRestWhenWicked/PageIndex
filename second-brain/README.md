# AI Second Brain

A personal knowledge-management system designed to be driven by Claude (Claude
Code, Claude Desktop, and an Obsidian vault). It combines the **PARA** method for
organizing knowledge with a **persistent memory layer** so an AI agent can pick
up context across sessions and help you capture, retrieve, and act on what you know.

> This vault is a *working scaffold*. Folders contain real, fillable files and
> templates rather than empty placeholders. Start by reading `CLAUDE.md` and
> `MEMORY.md`, then follow the daily workflow.

## What this gives you

- **A personal knowledge base** — everything you capture lives in one structured place.
- **AI-powered memory** — `CLAUDE.md` + `MEMORY.md` give the agent durable context.
- **Faster decisions** — retrieval and decision-support workflows on top of your notes.
- **Consistent workflows** — repeatable daily/weekly routines and reusable templates.
- **Compound intelligence** — value accrues as logs, notes, and memory build up.

## Layout

| Path | Purpose |
|------|---------|
| `CLAUDE.md` | Instructions the AI agent reads first every session. |
| `MEMORY.md` | Durable facts, decisions, and state the agent should remember. |
| `PARA/` | Projects · Areas · Resources · Archive · Inbox (core knowledge store). |
| `Daily/` | Daily captures and the morning-brief routine. |
| `Templates/` | Reusable note templates (capture, meeting, research, review, log). |
| `Workflows/` | Step-by-step routines the agent and you follow. |
| `Memory/` | Components that feed `MEMORY.md`: preferences, goals, AI instructions. |
| `Sessions/` | Per-session logs and continuity notes. |
| `Skills/` | Defined AI "skills" — repeatable, parameterized tasks. |
| `docs/` | Reference: setup, tool stack, best practices, pitfalls, outcomes. |

## Getting started

1. **Set up tools** — see `docs/setup-guide.md` and `docs/tool-stack.md`.
2. **Personalize memory** — fill in `Memory/preferences.md`, `Memory/goals.md`,
   then summarize the essentials into `MEMORY.md`.
3. **Run the daily loop** — `Workflows/daily-workflow.md`.
4. **Process your inbox** — `Workflows/capture-processing.md` (do this daily).
5. **Review weekly** — `Workflows/weekly-review.md`.

## Core principles

- **Log every session** so context is never lost.
- **Process the inbox daily** so capture stays cheap and nothing rots.
- **Keep notes structured** (use the templates) so retrieval stays reliable.
- **Maintain the memory files** so the AI stays aligned.
- **Build one skill per week** so the system compounds.

See `docs/best-practices.md` and `docs/pitfalls-to-avoid.md` for the full list.
