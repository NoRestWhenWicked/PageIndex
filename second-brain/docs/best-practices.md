# Best Practices

The habits that make the second brain actually work. Most are cheap; the payoff
compounds.

## Log every session
End each working session with a log in `Sessions/`. Context you don't write down is
context the next session has to rebuild. This is the single highest-leverage habit.

## Process the inbox daily
Capture should be frictionless *because* you trust that you'll sort it later. Empty
`PARA/Inbox/` every day (`Workflows/capture-processing.md`) and capture stays cheap.

## Keep notes structured
Use the templates. Consistent structure and clear titles are what make retrieval
reliable months later — both for you and for the agent searching on your behalf.

## Maintain the memory files
`MEMORY.md` is the agent's alignment. Update it whenever something durable changes.
A stale memory file quietly degrades every answer the agent gives.

## Build one skill weekly
Each week, teach the agent one new repeatable task (`Skills/`). This is how a static
note pile turns into a system that does more for you over time.

## Supporting habits
- **Link, don't duplicate** — one canonical note per thing.
- **Atomic notes** — one idea per note, clearly titled.
- **Review weekly** — the reset that keeps everything trustworthy.
- **Prune ruthlessly** — not every capture deserves to be kept.
