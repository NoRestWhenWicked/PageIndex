# Pitfalls to Avoid

The failure modes that quietly kill a second brain. Each one has a countermeasure
already built into this vault.

## Skipping session logs
Without logs, every session starts cold and context evaporates.
→ Always end with a `Sessions/` log. It's step one of the daily wind-down.

## Massive chat histories
Letting a single conversation balloon makes the agent slow and unfocused, and buries
decisions.
→ Keep sessions scoped; capture durable outcomes into `MEMORY.md` and a session log,
then start fresh.

## Unstructured notes
Freeform dumps are easy to write and impossible to retrieve.
→ Use `Templates/`; give notes clear titles and tags.

## Context overload
Stuffing everything into one prompt or one note dilutes signal.
→ Atomic notes, curated `MEMORY.md`, retrieval on demand instead of dumping it all in.

## Tool hopping
Spreading notes across many apps fragments your knowledge and breaks links.
→ One vault, many surfaces (`docs/tool-stack.md`). Capture everywhere, store in one place.

---
**Common thread:** these all trade a little discipline now for a lot of friction
later. The daily and weekly workflows exist to make the disciplined path the easy one.
