# Capture Processing (Inbox → PARA)

Run daily. The goal is an empty `PARA/Inbox/` and `Daily/Captures/` reviewed.
This is where raw capture becomes organized knowledge.

## For each captured item, decide:

1. **Is it actionable?**
   - **No** →
     - Useful reference? → `PARA/Resources/` (tag it).
     - Might need later? → `PARA/Archive/`.
     - Neither? → delete it. Not everything deserves to be kept.
   - **Yes** →
     - Has a deadline / defined outcome? → it belongs to a `PARA/Projects/` note.
     - Ongoing responsibility, no end date? → `PARA/Areas/`.
     - Single quick action? → add to today's task list, then discard the note.

2. **Does a home already exist?**
   - Link into the existing project/area note instead of creating a duplicate.

3. **Apply structure.**
   - Reformat into the right template if it's becoming a real note.
   - Give it a clear, searchable title and tags.

## Letting the agent help
Ask: *"Process my inbox."* The agent should propose a destination for each item
and apply your confirmations. It must **ask before deleting or overwriting**.

## Done when
- [ ] `PARA/Inbox/` is empty.
- [ ] Today's `Daily/Captures/` note is reviewed and its items routed.
- [ ] New tasks are on the task list; new reference notes are filed and tagged.
