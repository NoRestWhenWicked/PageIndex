# Weekly Review

A 30-45 minute reset that keeps the system trustworthy. Do it the same day each
week. Use `Templates/weekly-review.md` to capture the output in `Sessions/` (or a
dedicated `Reviews/` note).

## Steps

1. **Clear the decks** — process the inbox to zero; file stray captures.
2. **Reflect** — wins, slips, lessons. Be honest about what didn't happen.
3. **Review PARA** — for each project confirm it's still active and has a defined
   next action; demote finished/dead items to `Archive`; prune `Resources`.
4. **Goals check-in** — measure against `Memory/goals.md`; re-prioritize.
5. **Plan next week** — pick the top 3 priorities; resolve calendar conflicts.
6. **Maintain the system** —
   - Update `MEMORY.md`.
   - Refine one workflow or build one new `Skills/` skill.
   - Do a small knowledge cleanup (dedupe, re-tag, fix broken links).

## Why it matters
The weekly review is what turns a pile of notes into **compound intelligence**:
it keeps memory accurate, surfaces neglected areas, and feeds improvements back
into the workflows and skills the agent uses every day.
